# SQL Practice Exercises – BrightLearn Data Analytics Course

## Overview

This repository contains SQL practice exercises completed as part of the BrightLearn Data Analytics Course.

The goal of these exercises is to build a strong foundation in SQL by practicing the concepts most commonly used in data analysis and reporting. Each exercise focuses on understanding how data is stored, retrieved, transformed, and analyzed using SQL.

Rather than simply completing the exercises, I focused on understanding the logic behind every query and developing the ability to approach SQL problems with confidence.

---

## Topics Covered

### Selecting & Filtering

Learned how to retrieve specific information from a database using different filtering techniques.

Concepts covered:

* SELECT
* FROM
* WHERE
* DISTINCT
* ORDER BY
* LIMIT
* IN
* BETWEEN
* LIKE
* Comparison operators (`=`, `>`, `<`, `>=`, `<=`, `!=`)

---

### Aggregate Functions & Operators

Practiced summarising and analysing data using SQL aggregate functions and arithmetic operators.

Concepts covered:

* COUNT()
* SUM()
* AVG()
* MIN()
* MAX()
* Addition, subtraction, multiplication and division
* Percentage calculations
* Column aliases using `AS`

---

### CASE Statements

Applied conditional logic to classify and transform data into meaningful categories.

Concepts covered:

* CASE
* WHEN
* THEN
* ELSE
* END

Examples include:

* Stock status classification
* Attendance labels
* Salary bands
* Student grade categories

---

### Date Functions

Worked with date and time values to extract information and perform date calculations.

Concepts covered:

* CURRENT_DATE
* CURRENT_TIMESTAMP
* NOW()
* YEAR()
* MONTH()
* DAY()
* DAYNAME()
* DATEDIFF()
* TO_DATE()
* DATE_FORMAT()

---

### NULL Functions

Learned how to identify and handle missing values to produce cleaner and more reliable query results.

Concepts covered:

* IFNULL()
* COALESCE()
* Replacing NULL values with default values
* Managing incomplete data

---

## Learning Approach

Every query in this repository is written by hand as part of my learning process.

Instead of copying solutions, I worked through each exercise independently to understand the reasoning behind the SQL syntax and how different functions work together. This approach has helped me develop confidence in reading datasets, identifying the appropriate SQL techniques to use, and predicting what the output should look like before executing a query.

Writing the queries myself has also improved my ability to troubleshoot errors, think logically through problems, and build solutions step by step. I believe this hands-on approach has given me a much stronger understanding of SQL than simply memorising syntax.
